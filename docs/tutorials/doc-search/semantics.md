## Поиск документов по семантике

Для документов, охватывающих разные темы, можно использовать векторный семантический поиск. Процедура немного отличается от классического векторного поиска.

В исследовательской постановке цель — ранжировать документы, а не отдельные чанки: семантические совпадения агрегируются в документный сигнал. Это позволяет измерять качество отбора документов отдельно от качества последующего поиска по дереву.

### Пример пайплайна


#### Чанкинг и эмбеддинги
Разбейте документы на чанки, выберите модель эмбеддингов, чтобы преобразовать чанки в векторы, и сохраните каждый вектор вместе с соответствующим `doc_id` в векторной БД.

Исследовательский контроль здесь — размер чанков и стратегия перекрытия: они напрямую влияют на чувствительность к локальным признакам и шуму.

#### Векторный поиск

Для каждого запроса выполните векторный поиск и получите top-K чанков с их документами. 

Рекомендуется фиксировать K и сравнивать устойчивость ранжирования документов при разных моделях эмбеддингов, чтобы оценить вклад семантического слоя.

#### Расчет оценки документа

Для каждого документа вычислите релевантность. Пусть N — число чанков, относящихся к документу, а **ChunkScore**(n) — оценка релевантности чанка n. Оценка документа рассчитывается так:


$$
\text{DocScore}=\frac{1}{\sqrt{N+1}}\sum_{n=1}^N \text{ChunkScore}(n)
$$

- Сумма агрегирует релевантность по всем связанным чанкам.
- +1 под корнем гарантирует корректность формулы для документов с нулем чанков.
- Корень в знаменателе позволяет оценке расти с числом релевантных чанков, но с убывающей отдачей. Это поощряет документы с большим количеством релевантных чанков, но не дает крупным документам доминировать только из-за количества.
- Такая метрика предпочитает документы с меньшим числом, но более релевантных чанков, по сравнению с документами с большим числом слабых чанков.

В исследовательских экспериментах можно сравнить эту формулу с альтернативами (например, max/avg агрегацией), чтобы оценить компромисс между полнотой и точностью отбора документов.

#### Извлечение через PageIndex

Выберите документы с наибольшим DocScore, затем используйте их `doc_id` для дальнейшего извлечения через API PageIndex.
