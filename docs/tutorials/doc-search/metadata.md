

## Поиск документов по метаданным
<callout>Поддержка PageIndex с метаданными находится в закрытой бете. Заполните форму, чтобы запросить ранний доступ к этой функции.</callout>

Для документов, которые легко различать по метаданным, рекомендуем использовать метаданные для поиска.
Этот метод подходит для следующих типов документов:
- Финансовые отчеты, классифицированные по компании и периоду
- Юридические документы, классифицированные по типу дела
- Медицинские записи, классифицированные по пациенту или состоянию
- И многие другие

В таких случаях можно искать документы, используя их метаданные. Популярный метод — использовать "Query to SQL" для извлечения документов.

В исследовательской постановке метаданные выступают как высокоточный сигнал отбора: мы минимизируем шум, опираясь на структурированные поля, и оцениваем, насколько это улучшает итоговое извлечение по сравнению с семантическими подходами.

### Пример пайплайна

#### Генерация дерева PageIndex
Загрузите все документы в PageIndex, чтобы получить их `doc_id`.

#### Настройка SQL-таблиц

Сохраните документы вместе с их метаданными и `doc_id` PageIndex в таблице базы данных.

Качество эксперимента здесь зависит от консистентности схемы и полноты метаданных, поэтому полезно фиксировать правила нормализации и справочники значений.

#### Query to SQL

Используйте LLM, чтобы преобразовать запрос пользователя в SQL-запрос для выборки релевантных документов.

Для исследовательской оценки можно сравнивать точность генерации SQL и полноту отбора документов на контролируемом наборе запросов.

#### Извлечение через PageIndex

Используйте `doc_id` PageIndex для выбранных документов, чтобы выполнить дальнейшее извлечение через API PageIndex.
