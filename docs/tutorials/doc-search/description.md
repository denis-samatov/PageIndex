
## Поиск документов по описанию

Для документов без метаданных можно использовать описания, сгенерированные LLM, чтобы помочь в выборе документа. Это легковесный подход, который лучше всего работает на небольшом числе документов.

В исследовательской рамке описания выступают как прокси-сигнал содержания: мы проверяем, насколько краткая репрезентация документа позволяет отобрать релевантные источники до запуска полного поиска по дереву. Это особенно полезно для быстрой оценки качества коллекции без сложной инфраструктуры.

### Пример пайплайна


#### Генерация дерева PageIndex
Загрузите все документы в PageIndex, чтобы получить их `doc_id` и структуру дерева.

#### Генерация описания

Сгенерируйте описание для каждого документа на основе дерева PageIndex и кратких описаний узлов.
```python
prompt = f"""
Вам дана структура оглавления документа. 
Ваша задача — сгенерировать одно предложение-описание, которое помогает отличить документ от других.
    
Древовидная структура документа: {PageIndex_Tree}

Верните только описание, без дополнительного текста.
"""
```
Для экспериментального контроля важно фиксировать стиль и длину описаний, чтобы сравнение между документами было корректным.

#### Поиск с помощью LLM

Используйте LLM, чтобы выбрать релевантные документы, сравнивая запрос пользователя с сгенерированными описаниями.

В исследованиях обычно измеряют точность отбора на небольших выборках вручную размеченных запросов и анализируют ошибки, связанные с недостаточной детализацией описаний.
Ниже пример промпта для выбора документов по их описаниям:

```python
prompt = f""" 
Вам дан список документов с их идентификаторами, именами файлов и описаниями. Ваша задача — выбрать документы, которые могут содержать информацию, релевантную пользовательскому запросу.

Запрос: {query}

Документы: [
    {
        "doc_id": "xxx",
        "doc_name": "xxx",
        "doc_description": "xxx"
    }
]

Формат ответа:
{
    "thinking": "<Ваше рассуждение о выборе документов>",
    "answer": <Список релевантных doc_id в формате Python>, например ['doc_id1', 'doc_id2']. Верните [] если релевантных документов нет.
}

Верните только JSON-структуру без дополнительного текста.
"""
```

#### Извлечение через PageIndex

Используйте `doc_id` PageIndex для выбранных документов, чтобы выполнить дальнейшее извлечение через API PageIndex.
