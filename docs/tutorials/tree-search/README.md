## Примеры поиска по дереву
Этот туториал показывает базовый пример извлечения с использованием дерева PageIndex.

В исследовательской постановке дерево трактуется как поисковое пространство, а задача извлечения — как выбор минимального набора узлов, покрывающего релевантные разделы с приемлемым балансом полноты и точности. Такой взгляд позволяет явно отделять ошибки навигации по структуре от ошибок генерации ответа.

### Базовый пример поиска по дереву с LLM
Простая стратегия — использовать LLM-агента для обхода дерева. Ниже базовый промпт.

```python
prompt = f"""
Вам даны запрос и древовидная структура документа.
Нужно найти все узлы, которые, вероятно, содержат ответ.

Запрос: {query}

Древовидная структура документа: {PageIndex_Tree}

Ответьте в следующем JSON-формате:
{
  "thinking": <ваше рассуждение о том, какие узлы релевантны>,
  "node_list": [node_id1, node_id2, ...]
}
"""
```
В экспериментальной оценке важно фиксировать размер `node_list` и сравнивать полноту покрытия ответных страниц с альтернативными стратегиями, например жадным обходом по заголовкам или ручной разметкой.
<callout>
В нашем дашборде и API извлечения мы используем комбинацию LLM-поиска по дереву и MCTS на основе функции ценности ([MCTS](https://en.wikipedia.org/wiki/Monte_Carlo_tree_search)). Подробности скоро будут опубликованы.
</callout>

### Интеграция предпочтений пользователя или экспертных знаний
В отличие от векторного RAG, где интеграция экспертных знаний или пользовательских предпочтений требует дообучения модели эмбеддингов, в PageIndex вы можете добавить предпочтения или экспертные знания просто в промпт LLM-поиска по дереву. Ниже пример пайплайна.

С исследовательской точки зрения это способ вводить априорные гипотезы о релевантных разделах и измерять их влияние на точность выбора узлов, не меняя базовую структуру индекса.

#### 1. Извлечение предпочтений

Когда поступает запрос, система выбирает наиболее релевантные фрагменты пользовательских предпочтений или экспертных знаний из базы данных или набора доменных правил. Это можно сделать через поиск по ключевым словам, семантическому сходству или LLM-поиск релевантности.

#### 2. Поиск по дереву с учетом предпочтений
Интегрируем предпочтения в промпт поиска по дереву.

**Пример усиленного поиска по дереву с учетом экспертных предпочтений**

```python
prompt = f"""
Вам даны вопрос и древовидная структура документа.
Нужно найти все узлы, которые, вероятно, содержат ответ.

Запрос: {query}

Древовидная структура документа:  {PageIndex_Tree}

Экспертные знания о релевантных разделах: {Preference}

Ответьте в следующем JSON-формате:
{
  "thinking": <рассуждение о том, какие узлы релевантны>,
  "node_list": [node_id1, node_id2, ...]
}
"""
```

**Пример экспертного предпочтения**
> Если запрос упоминает корректировки EBITDA, приоритетно рассматривайте Item 7 (MD&A) и примечания в Item 8 (Financial Statements) в отчетах 10-K.

Интеграция пользовательских или экспертных предпочтений делает поиск по узлам более точным и эффективным, используя как структуру документа, так и доменные знания.
